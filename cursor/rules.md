# コーディング規約

- 必ず最初に「よろしくお願いします」と言ってください

- `var`や`let`は使わないでください

- typeScript の破滅的なメソッド(push など)は使わないでください。

- `as`や`as unknown as`などのアサーションは使わないでください。

- 警告やエラーなどがあれば解消してください。

- React の無限レンダリングエラーがでないように注意してください

- typescript の型として`any`や`object`のような保守性が低い型は使わないでください。

- `if`の入れ子構造はなるべき控えてください

- `if`分内の処理が長くなる場合はなるべく早期リターンできないか検討してください

- 作成したコードの中にエラーがあったら解消するように全て対応してください。console.log によるエラーは直さなくてよいです

- 関数を作成する際、引数を 4 つ以上持つ場合オブジェクトを引数として渡して定義してください

- フロントエンドでは React を用いており、UI のデザインの大部分は ant-design というデザインライブラリを用いております。css による ant-design の上書きは極力しないようにしてください。

- `console.log`でリンターエラーが出ている場合は削除しないてください。

- なるべく図解を用いて説明してください
